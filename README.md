# CS 230 Portfolio Reflection

The case study company, The Gaming Room, created a game called Draw It or Lose It that was originally only available on Android. The project goal was to design a web based version that could work across multiple platforms. The application needed to support multiple games, teams, and players while keeping data organized and preventing duplicate game or team names. I connected these requirements to the design document, UML diagram, and provided Java code. I also completed the required sections in classes such as GameService, ProgramDriver, and SingletonTester. This helped me understand how the singleton pattern could manage one GameService instance and how iteration could check existing games for duplicate names.

The design document helped me think through the structure of the program before focusing on implementation. It made concepts like inheritance, encapsulation, and design patterns easier to understand because I could connect them to the actual class structure. If I could revise one part of my work, I would improve the explanation of the design patterns in my code comments and documentation. I would make the purpose of the singleton and iterator patterns clearer so another developer could understand why those choices were used in the application.

I interpreted the user’s needs by focusing on the case study requirements throughout the course. The game needed to move beyond Android, support multiple platforms, manage multiple games, teams, and players, and keep names unique. Those requirements guided the UML diagram, code updates, design document, platform comparison, and final recommendations. My approach to designing the software was to break the system into smaller parts and determine how those parts should connect. In the future, I would use the same approach by reviewing requirements, creating a clear model, applying useful design patterns, and comparing architecture options before choosing the best fit.
